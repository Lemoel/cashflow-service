# Idiomas: código em inglês, mensagens de UI em português

## Inglês (parte de programação)

Usar inglês em toda a parte de programação:

- Classes, métodos, campos (properties), variáveis, parâmetros.
- Nomes de endpoints (paths), tabelas e colunas em migrations.
- Mensagens de log, nomes de testes, identificadores em geral.

## Português (mensagens de interface)

Usar português em mensagens de interface com o usuário:

- Textos exibidos na UI (labels, botões, mensagens de erro/sucesso, placeholders, títulos de página).
- Mensagens de validação exibidas ao usuário no frontend ou retornadas pela API para exibição na tela.
